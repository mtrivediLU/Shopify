Technical Thinking

How I would think about a merchant integration at Shopify.

This is a simplified architecture view showing how I would approach reliability, API boundaries, observability, and merchant outcomes.

ERP OMS PIM Fulfillment Payments
Integration App Auth · Webhooks · Queues · Retries · Rate Limits · Logging Built for merchant launch timelines and production support.
Admin API Storefront API Custom UI Monitoring Runbooks

Engineering decisions I would make early

  • Clarify business event ownership before coding the integration.
  • Separate synchronous user flows from asynchronous background jobs.
  • Design idempotency and retry behavior before launch.
  • Respect API rate limits with queues and backoff logic.
  • Expose clean merchant status messages, not only logs.
  • Document support paths and failure modes for launch readiness.
